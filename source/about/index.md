### 核心项目成就

#### 1. WMS 仓储系统千万级数据性能重构 (S级项目)
* **项目背景**：服务于头部客户（上海文信），库存表数据量突破 **3000万**。在生产高峰期，PDA 入库操作响应长达 **30秒**，频发死锁与超时，严重影响产线节拍。
* **解决方案**：
    * **计算下推架构**：针对工厂弱网环境，重构通信机制。利用 **SQL Server 存储过程** 将复杂的 Java 业务逻辑下沉至数据库层，减少 90% 的网络 IO 交互。
    * **并发冲突治理**：设计**“乐观锁 + 分段事务”**机制，替代原有的长事务全表锁；引入 `NOLOCK` 读写分离策略，彻底解决多人操作同一库位时的死锁问题。
    * **SQL 执行优化**：通过执行计划分析，重构高频查询的索引策略与 Join 逻辑，消除全表扫描。
* **成果**：将入库接口平均响应时间从 **30秒锐减至 500毫秒**（性能提升 60 倍），彻底解决了现场 PDA “转圈卡死”的客诉问题。

#### 2. 智能余料复用推荐系统 (成本优化)
* **项目背景**：针对生产过程中产生的高价值板材余料缺乏管理、利用率低的问题，设计智能回收与推荐机制。
* **解决方案**：
    * **算法迁移与适配**：基于三维装箱算法的底层逻辑，结合板材加工特性（纹理方向、最小夹持边距），实现了**“板材余料智能匹配推荐功能”**的工程化落地。
    * **高性能推荐**：利用 Redis 缓存余料特征指纹，在工人在切料工位扫码的瞬间，500ms 内匹配出最优的库存余料方案，替代新板开料。
* **成果**：显著提升了板材利用率，大幅降低了原材料浪费，成为公司软件在高定工厂的核心卖点之一。

#### 3. 佛山xxx PDA 扫码全链路优化
* **项目背景**：客户现场 PDA 扫描工件响应慢（10秒+），且经常出现漏扫、重扫数据不一致。
* **解决方案**：
    * **异步化改造**：将原本同步的数据库校验重构为 **Event-Driven（事件驱动）** 模型，利用多线程并行处理数据。
    * **边缘侧缓存**：在 PDA 端引入本地 SQLite/缓存机制，实现“离线扫描、断网续传”，保障网络抖动时不丢数据。
* **成果**：将扫码响应速度优化至 **1秒以内**，支持产线工人的连续快速作业。

#### 4. 新一代 MES 架构升级与工程效能建设
* **CI/CD 体系搭建**：从零搭建基于 **Jenkins + Docker** 的自动化构建与部署流水线。将 MES 服务端与 PDA 程序的发版时间从小时级缩短至 **5分钟**，实现了“代码提交即构建”，杜绝了人工部署导致的配置错误。
* **IIoT 数据平台预研**：针对 InfluxDB 在高基数（千万级 SN 码）场景下的查询瓶颈，设计了基于 **ClickHouse** 的新一代日志与时序数据存储架构，并完成了 MQTT 高并发接入的压测验证。
